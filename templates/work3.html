003 : EMAIL SENDER

This project is all about sending a mail using python code

This project is built using python language and it's module called as twilio

Scroll below for screenshots


Importing Modules:

The script begins by importing necessary Python modules:

  • smtplib: for handling the SMTP (Simple Mail Transfer Protocol) communication.
  • EmailMessage: for creating and managing email messages.
  • Template: for string templating using placeholders.
  • Path: for working with file paths.
  • sys: for accessing command-line arguments.

Reading HTML Template:

The script reads the content of an HTML file named index.html using the Path module. The content is then treated as a template using the Template module, allowing for variable substitution.

Creating Email Message:

An EmailMessage object is created to represent the email. The sender is set to 'Python', the recipient is set to the email address provided as a command-line argument (sys.argv[1]). The email subject is set to a predefined string. The HTML content of the email is set by substituting a placeholder ({name}) in the template with the value provided as the second command-line argument (sys.argv[2]).

SMTP Connection:

A connection to the Gmail SMTP server is established using the smtplib.SMTP class. The ehlo() method is called to greet the server, and starttls() is used to initiate a secure connection. The login method is then used to authenticate with the Gmail account by providing the email address and password directly in the script.

Sending Email:

The prepared email message is sent using the send_message method of the SMTP connection.

Print Success Message:

If the email is sent successfully, the script prints "All good" to indicate that the process completed without errors.

Important notes:

The script relies on command-line arguments, specifically the recipient's email address (sys.argv[1]) and a name for substitution in the HTML template (sys.argv[2]).Security concerns include storing sensitive information (Gmail credentials) directly in the script. Best practices involve using environment variables or a configuration file for such data.
